# 타임시트 권한 제어 및 주간 단위 조회 최적화 결과 보고서

타임시트(Timesheet) 메뉴 진입 시 방대한 전체 데이터(5만여 건) 조회로 인해 발생하는 지연 현상을 최적화하고, 보안 정책을 개선하기 위해 조회 구조 개편 및 권한 제어를 적용했습니다.

## 변경 사항

### Backend

- [api_router.py](file:///f:/pe/public_html/steelworks-manager-react/core/api_router.py)
  - `[NEW] /api/punch/latest_week` 엔드포인트를 신설하여 가장 최근의 펀치 데이터가 존재하는 연도(`year`)와 주차(`week`)를 고속으로 조회할 수 있는 기능을 추가했습니다.
  - `/api/punch/timesheet` 엔드포인트를 개선했습니다:
    - 99레벨(슈퍼 어드민, 즉 admin ID)만 전체 조회 및 타인 검색이 가능하게 하고, 그 미만의 권한(Ulrich 등 level 12 포함)은 본인 데이터만 볼 수 있도록 조치하여 보안을 강화했습니다.
    - `year`, `week` 쿼리 매개변수를 처리해 데이터베이스 검색 레벨에서 색인화 쿼리를 사용하여 5만 건 스캔 없이 고속으로 검색하도록 최적화했습니다.
    - 필터링된 주간 데이터에 대해 요일(월요일~일요일) 순으로 1차 정렬한 뒤, 동일 요일 내에서는 펀치 ID의 내림차순(최신순)으로 정렬해 정렬 순서를 교정했습니다.
    - 데이터가 적어짐에 따라 병목을 줄이기 위해 파이썬 단의 문자열 매핑 지연을 최소화했습니다.

### Frontend

- [page.js (Timesheet)](file:///f:/pe/public_html/steelworks-manager-react/fe/src/app/dashboard/timesheet/page.js)
  - `isAdmin` 변수 판정을 `user?.right_level === 99`로 수정하여 슈퍼 어드민만 관리자 드롭다운을 제어하도록 구성했습니다.
  - 시작일/종료일 선택창을 연도(Year) 및 주차(Week) 입력 필터로 완전히 전환했습니다.
  - `devHints` 옵션이 설정되어 있는 경우 노출되는 `Auto Week (Dev)` 임시 검색 버튼을 신설했습니다. 이를 통해 DB에 기록된 가장 최신 주간(예: 2021년 46주)의 펀치 데이터가 있는 주간을 자동으로 탐색하여 세팅해 줍니다.

## 확인 및 검증 결과

- 관리자(Ulrich 등)로 로그인 시 본인 데이터만 제한되어 초고속으로 조회됩니다.
- 개발 환경의 테스트 데이터가 가장 최신 주간인 경우에도 `Auto Week (Dev)` 버튼 클릭 시 0ms대의 고속 조회와 정렬(월~일 오름차순, 요일 내 ID 내림차순)이 완벽하게 보장됩니다.
