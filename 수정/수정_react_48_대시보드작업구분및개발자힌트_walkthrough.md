# 대시보드 실시간 작업 구분 표시 및 공통 개발자 힌트 기능 구현 결과서 (Walkthrough)

대시보드 작업 목록 개선 및 화면별 개발자 힌트 표시 기능이 성공적으로 구현되었습니다.

## 구현 및 변경 사항

### 1. 대시보드 실시간 작업 구분 및 나란히 배치 (`fe/src/app/dashboard/page.js`)
- 대시보드의 실시간 작업 진행률 목록을 **In Production Jobs (진행 중인 작업)**과 **Completed Jobs (완료된 작업)** 두 개 패널로 분리했습니다.
- 두 패널을 데스크톱 화면에서 **나란히(Side-by-Side)** 배치하여 한 눈에 볼 수 있도록 구성했습니다.
- 진행 중인 작업은 파란색 바(Blue), 완료된 작업은 초록색 바(Green)를 표시하도록 통일했습니다.
- 각 패널당 최대 50개의 작업이 노출되도록 제한하였으며, 데이터가 초과할 시 개별적으로 작동하는 **이전/다음 페이지네이션** 기능을 제공합니다.
- 정렬 기준은 동일하게 Job # 오름차순(ascending)입니다.

### 2. 공통 DevHints 컴포넌트 추가 (`fe/src/components/common/DevHints.js`)
- `SHOW_DEV_HINTS = True` 설정이 되어 있을 때, 화면 하단에 현재 페이지와 관련된 Frontend 파일, Backend 파일 및 API, DB 테이블 및 상세 조작 조건을 이쁘게 보여주는 컴포넌트를 개발했습니다.
- 이모지나 특수 문자를 배제한 텍스트 기반의 반응형 디자인을 적용하여 빌드 에러를 방지했습니다.

### 3. DashboardLayout 적용 (`fe/src/app/dashboard/layout.js`)
- `DashboardLayout`의 `main` 콘텐츠 영역 하단에 `DevHints` 컴포넌트를 주입하여, 대시보드 하위의 모든 메뉴 페이지에서 자동으로 개발자 힌트가 출력되도록 수정했습니다.

## 검증 결과

- **빌드 테스트**: `npm run build` 결과 에러 없이 성공적으로 빌드가 완료되었습니다.
- **기능 검증**: 백엔드의 `SHOW_DEV_HINTS` 설정에 따라 개발자 힌트 카드가 올바르게 출력 및 숨김 처리되는 것을 확인했습니다.
