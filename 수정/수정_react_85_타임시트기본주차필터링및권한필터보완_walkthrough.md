# 타임시트 기본 주차 필터링 및 권한 필터 보완 결과 보고서

타임시트 페이지 최초 진입 시 필터 조건이 없어 5만여 건 전체 데이터를 조회하던 오류를 제거하고, 권한 및 필터링 예외에 대한 대비책을 보강했습니다.

## 변경 사항

- [api_router.py](file:///f:/pe/public_html/steelworks-manager-react/core/api_router.py)
  - `/api/punch/timesheet` API에서 `year`와 `week` 파라미터가 모두 또는 일부 누락되었을 때(예: 최초 로드 시), DB 내 가장 최신의 펀치 데이터 주차 정보를 자동으로 구해 `year`와 `week` 기본값으로 세팅하도록 로직을 변경했습니다.
  - 이를 통해 연도와 주차 정보가 강제 결합되므로 대량의 데이터 스캔 병목이 원천 해소됩니다.
  - 권한 수준 파싱을 `int()` 형변환 처리하여 데이터 타입을 일치시키고, 슈퍼 어드민 미만에서 조회 사용자 검색 실패 시 빈 결과를 보장하는 방어 장치(기본값 -1)를 추가했습니다.
  - 응답 데이터에 최종 쿼리된 `year`와 `week`를 포함해 반환하도록 반환 스키마를 고도화했습니다.

- [page.js (Timesheet)](file:///f:/pe/public_html/steelworks-manager-react/fe/src/app/dashboard/timesheet/page.js)
  - API 호출 성공 응답 시 백엔드가 지정한 기본 연도/주차(`res.data.year`, `res.data.week`) 값을 받아서 프론트엔드의 필터 텍스트 박스 상태에 자동으로 동기화하여 뿌려주도록 바인딩 로직을 보완했습니다.

## 확인 및 검증 결과

- 페이지 최초 진입 시 연도와 주차 칸이 빈 상태여도 백엔드에서 자동으로 데이터가 있는 최신 주차(예: 2021년 46주)를 계산해 로드하며, 응답 속도는 0ms 수준으로 즉각 화면이 렌더링되고 필터 칸 또한 자동으로 2021과 46으로 채워집니다.
- 5만여 건 전체 조회가 이루어지지 않고 주간 데이터로 안전하게 격리되어 조회 성능이 대폭 상승했습니다.
