# 개발 테스트용 퀵필 고유 평문 노출 및 해시 보안 복구 결과서

## 1. 조치 사항
- **보안 취약점 롤백**:
  * 해시 문자열을 1:1로 직접 검증해서 무조건 통과시키던 로그인 API 바이패스 조건(`payload.password == user['password']`)을 완벽히 제거하여 상용 운영 시 보안성을 정상 복구했습니다.
- **계정별 고유 비밀번호 일괄 리셋 개발 (`skills/200_admin_pipeline.py`)**:
  * `Reset to Unique Hashed` 실행 시, 모든 사용자의 초기 패스워드를 동일한 값이 아닌 `'dev_[사용자ID]'` (예: `dev_Aaron`, `dev_Alison` 등) 형태로 개별 다르게 할당하여 `bcrypt` 해싱 후 DB에 일괄 저장하도록 로직을 변경했습니다.
- **백엔드 매핑 검출을 통한 퀵필 실시간 평문 노출**:
  * `/api/employees` API 로드 시 각 사용자의 DB 비밀번호 해시값을 검사하여, `'12345678'`의 해시인지 혹은 `'dev_[login]'`의 해시인지 백엔드에서 `bcrypt.checkpw`로 미리 점검합니다.
  * 일치 판정 시 해당 평문 원본 문자열(`12345678` 또는 `dev_Aaron` 등)을 프론트엔드로 실시간 역전송하고, 그 외 사용자가 변경한 세션 패스워드는 해시값 원문 그대로 퀵필에 노출합니다.
  * 사용자가 퀵필을 클릭하면 노출된 고유 평문 텍스트(`dev_Aaron` 등)가 로그인 폼에 자동 입력되며, 로그인 시 정상적으로 `bcrypt.checkpw` 암호 연산을 태우고 대시보드 진입에 성공하게 됩니다.
