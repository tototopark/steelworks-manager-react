# 구현 계획서 - README.md 작성 및 GitHub 배포 준비

이 계획서는 포트폴리오의 완성도를 극대화하기 위해 프로젝트 루트 경로에 대외용 `README.md` 파일을 작성하고, Git 저장소 업로드를 준비하는 과정을 다룹니다. 특히 62세 철골 제작 엔지니어로서 AI 협업(Vibe Coding)을 통해 24시간 만에 레거시 마이그레이션 포트폴리오를 구축한 혁신적인 스토리를 리드미에 녹여내어 채용 담당자에게 강한 인상을 남기도록 구성합니다.

## 사용자 검토 요구사항

> [!IMPORTANT]
> - **스토리텔링 반영**: 뉴질랜드 경제 악화로 인한 인원 감축 이후, 도메인 지식(20년 이상의 철골 제작/설계 경험)과 AI 바이브 코딩 기술을 융합하여 단기간에 완성한 포트폴리오임을 강조합니다.
> - **기술 스택 시각화**: Next.js App Router + FastAPI + SQLite 스택을 명확히 명시하고 주요 기능(QA WIP, Workload, Timesheet 등 14개 기능)의 작동 방식을 설명합니다.
> - **실행 방법 명시**: 백엔드와 프론트엔드를 손쉽게 가동할 수 있는 윈도우 배치파일(`r.bat`) 실행 방법 등을 명시합니다.
> - **이모지 사용 안함**: 시스템 규칙에 따라 리드미와 계획서 문서 전반에 이모지 사용을 금지합니다.
