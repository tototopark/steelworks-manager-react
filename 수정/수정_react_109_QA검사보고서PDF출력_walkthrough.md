# 수정_react_109_QA검사보고서PDF출력_walkthrough

## 개요

레거시 앱의 `73.php` (QA Report Print) 기능을 2번 앱(steelworks-manager-react)에 품질 보증 검사 성적서 팝업창 조립 및 브라우저 인쇄 트리거 방식으로 구현하였습니다.

## 구현 내용

### 1. 프론트엔드 - qa-wip/page.js
- **Printer 아이콘 연동:**
  - `lucide-react` 라이브러리의 `Printer` 아이콘을 임포트하여 직관적인 UI 인터페이스를 구성하였습니다.
- **QA 보고서 인쇄용 마크업 및 스타일링 신설 (`handlePrintQAReport`):**
  - 특정 잡을 선택했을 때, 현재 QA WIP 테이블 목록에 노출되는 미검수/재작업 진행 중인 부재 리스트 정보를 모아서 A4 표준 양식에 맞춘 HTML/CSS 레이아웃을 조립합니다.
  - **포함 정보**: 검사 대상 잡 번호, 거래처명, 출력일, 총 검사 대상 수량, 부재 리스트 테이블(LOT, 부재마크, WPS 공법 코드, 담당 검사원, 재작업 버전, NCR 불합격 상세 사유) 및 최종 서명란.
  - `@media print` 쿼리를 적용하여 인쇄 시 웹 브라우저 UI는 마스킹되고 순수 성적서 테이블만 용지에 인화되거나 PDF로 저장될 수 있도록 구현했습니다.
  - 새 창 팝업 호출 후 즉시 `window.print()`를 트리거하여 인쇄 작업을 간소화하였습니다.
- **인쇄 실행 단추 배치:**
  - QA WIP 페이지의 우측 상세 WIP Items 헤더 패널의 수량 배지 바로 우측에 **"Print Report"** 버튼을 배치하고 이벤트 리스너를 바인딩하였습니다.
