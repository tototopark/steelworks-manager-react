# 타임시트 개인 조회 강제 및 전체 조회 차단 결과 보고서

관리자(level 99) 로그인 시 발생했던 대량의 전체 펀치 내역(1주치 1,017건) 로드 병목을 해결하고, 날짜 영역에 요일을 추가 표기하도록 개선했습니다.

## 변경 사항

- [api_router.py](file:///f:/pe/public_html/steelworks-manager-react/core/api_router.py)
  - `/api/punch/timesheet` API에서 `query_emp_id`가 지정되지 않았거나(어드민 최초 진입 시 등), 슈퍼 어드민이 아님에도 타인의 데이터를 요청하는 등 방어 조건에 걸리면 데이터를 쿼리하지 않고 빈 배열(`[]`)을 즉시 반환하도록 조치했습니다.
  - 마운트 시 호출되는 `/api/punch/latest_week` API에서 99레벨 미만(Ulrich 등 level 12 포함) 사용자는 본인의 최근 펀치 기록을 기준으로 가장 최신 데이터의 연도와 주차를 추적하도록 쿼리를 보완했습니다.
  - 조회되는 모든 펀치 로그의 날짜 데이터에 대해 요일 정보(`day_of_week` 예: "월", "화"...) 필드를 백엔드 단에서 요일 인덱스를 기준으로 매핑하여 제공하도록 구현했습니다.

- [page.js (Timesheet)](file:///f:/pe/public_html/steelworks-manager-react/fe/src/app/dashboard/timesheet/page.js)
  - 어드민의 직원 선택 드롭다운 select 박스에서 `All Employees` 옵션을 완전히 제거하여 전체 조회를 원천 방지하고 1인 단위 조회만 강제하도록 구조를 바꿨습니다.
  - 컴포넌트 마운트 및 직원 목록 로드 성공 시, `selectedEmp`가 빈 값이면 자동으로 리스트의 첫 번째 직원 ID를 지정하여 최초 데이터 쿼리가 성공적으로 이루어지도록 로직을 보완했습니다.
  - 테이블 로그 목록 렌더링 시 날짜 컬럼의 값 옆에 요일 정보(예: `2021-07-06 (화)`)가 괄호로 노출되도록 렌더링을 개선했습니다.

## 확인 및 검증 결과

- 관리자(admin)로 로그인하여 타임시트 진입 시 전체 1,017건이 쏟아져 나오지 않고, 목록의 첫 번째 직원이 기본 지정되어 해당 직원의 최신 1주치(수십 건 수준) 기록만 0ms 수준으로 즉시 가볍게 로드됩니다.
- 날짜 옆에 한글 요일 표시가 정확하고 깔끔하게 렌더링되는 상태를 검증했습니다.
- 일반 사용자(`Ulrich` 등)로 로그인하여 `Auto Week (Dev)` 버튼 클릭 시 본인의 데이터가 위치한 최신 주간(예: 2021년 46주 등)으로 안전하게 탐색 및 세팅되어 펀치 데이터가 정상적으로 검색됩니다.
