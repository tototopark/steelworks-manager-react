[Steelworks Manager React - 마이그레이션 및 세션 인계 확인사항]

1. 개발 환경 정보
- 대상 앱 위치: F:\pe\public_html\steelworks-manager-react
- 백엔드 실행: python run_api.py (FastAPI, 포트 3600)
- 프론트엔드 실행: npm run dev (Next.js, 포트 3001)

2. 현재까지 완료된 마이그레이션 작업 (기완성)
- 로그인 및 대시보드 레이아웃: 다크 테마(Zinc-950) 기반 구성 및 백엔드 세션 연동 완료
- Monthly Plan (주간/월간 일정): 캘린더 그리드 및 일별 메모, 생산 계획 연동 완료
- Whiteboard (화이트보드): HTML5 드래그앤드롭 기반 직원 작업 배정 칸반 보드 구축 완료
- QA WIP (품질 검사): 품질 판정 프로세스 및 불합격 시 자동 재작업 지시 연동 완료
- Employees (직원 관리): 아바타 이미지 업로드 및 임시 패스워드 발급기 연동 완료
- Vehicles (차량 관리): WOF/REGO 만료일 기준 30일 이내 알림 모듈 완료
- Admin DB (어드민 DB 브라우저): Level 99 권한 제한이 적용된 SQLite 테이블 데이터 뷰어 및 DB 초기화/시딩 툴킷 완료
- Timesheet (작업 시간 일지): 펀치 로그 조회 및 권한 필터링(관리자는 전체, 일반 직원은 본인 정보만 조회), CSV 다운로드 기능 추가 완료
- 부재별 가공 시간 추적 버튼 추가: Jobs 상세 화면의 부재 목록 우측에 Start/Stop 버튼을 추가하여 실제 제작 펀치 API 연동 완료
- Punch Clock (근태 기록): 대리 출근 방지를 위해 로그인한 사용자 세션을 바인딩하여 CLOCK IN/OUT 처리하도록 개편 완료

3. 미마이그레이션 상태 및 보완/향후 진행할 작업 정리
- Holidays (공휴일 관리) 화면 개발 필요
  * 백엔드 API (/api/holidays GET, POST, DELETE)는 준비되어 있으나 프론트엔드 UI 페이지(fe/src/app/dashboard/holidays/page.js)가 없습니다.
  * 사이드바 메뉴에도 Holidays가 누락되어 있어 Level 10+ 관리자만 접근할 수 있도록 가드를 적용한 공휴일 일정 등록/삭제 페이지를 신규 생성하고 링크해야 합니다.
- Jobs 상세 뷰 내 도면 및 현장 사진 관리 컴포넌트 추가 필요
  * 백엔드 API (/api/jobs/{job_number}/photos GET, POST)는 완성되어 있으나 프론트엔드 Jobs 상세 화면에 사진/도면 업로드 및 아카이브 갤러리 컴포넌트가 누락되어 있습니다.
- 직원 마스터 IP 접근 제한 복원 필요
  * 레거시(sitepro - 11.php / 12.php)에 있던 기기 접근 제어(IP_1, IP_2, IP_3 등록 및 매칭 검증) 기능이 React 앱의 직원 관리 및 백엔드 인증 미들웨어에 미이식 상태입니다.
- 그리드 일괄 편집 기능 (Job Sheet Edit) 마이그레이션 필요
  * 레거시의 100라인 단위 일괄 편집 그리드(Member, Lot, Galv/Paint, 공수 정보 일괄 수정 및 엑셀 가져오기)가 2번 앱(React)의 Jobs 화면에 아직 포팅되지 않았습니다.
- 대시보드 진행바 시각적 동기화 및 페이징 보완 필요
  * 현재 2번 앱 대시보드는 최신 5개만 정렬 없이 단순 화이트 진행바를 출력하고 있습니다.
  * 이를 1번 앱 디자인처럼 진행률에 따른 포인트 컬러(진행 중: 블루, 완료: 그린)와 진행바 내부의 완료 수량 분수비(예: 85% (17/20 pcs)) 오버레이가 들어가도록 수정해야 합니다.
  * 또한 활성 작업을 동시에 최대 50개까지 Job Number 오름차순으로 보여주고, 50개 이상일 경우 페이징 처리가 되도록 해야 합니다.
- 비밀번호 변경 (Change Password) UI 보완 필요
  * 기본 패스워드인 '12345678'로 로그인할 경우 토큰에 require_change 플래그를 담아 반환하나, 프론트엔드에서 강제 비밀번호 변경을 수행할 수 있는 독립 UI 페이지나 모달이 명확히 연결되어 있지 않으므로 이에 대한 확인 및 추가 구현이 요구됩니다.

이 체크리스트와 진행 상황 요약을 바탕으로 다음 세션에서 작업을 즉시 이어서 수행할 수 있습니다.
