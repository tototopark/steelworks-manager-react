# 웹 서비스 실제 사용을 위한 배포 가이드

이 문서는 완성된 Steelworks Manager (React + FastAPI) 시스템을 실제 환경에서 가동하고 채용 담당자나 동료들이 직접 접속하여 사용해 볼 수 있도록 배포하는 구체적인 방안을 설명합니다.

본 프로젝트는 Next.js 프론트엔드가 프로덕션 빌드되어 정적 파일로 컴파일된 후, FastAPI 백엔드 파이썬 서버가 이 정적 리소스를 직접 서빙하도록 통합 설계되어 있습니다. 따라서 프론트엔드와 백엔드를 별도로 배포할 필요 없이 단 하나의 파이썬 웹 서비스 인스턴스만 띄우면 되므로 배포 비용이 매우 저렴하고 구조가 단순합니다.

---

## 1. 사내 로컬 네트워크 배포 (가장 간단하고 실무에 유용한 방식)

사무실이나 특정 공간의 로컬 네트워크(Wi-Fi 또는 LAN) 내에서 여러 PC와 디바이스(태블릿, 스마트폰)로 시스템을 함께 사용하고 싶을 때 적합합니다.

### Step 1. 프론트엔드 정적 파일 빌드
Next.js 최적화 빌드를 수행하여 static 자원들을 생성합니다.
1. 터미널을 열고 `fe` 폴더로 이동합니다.
2. 아래 명령을 실행합니다:
   ```bash
   cd fe
   npm run build
   ```
3. 빌드가 완료되면 static 폴더 안에 컴파일된 프론트엔드 자원들이 적재됩니다.

### Step 2. 백엔드 서버 로컬 실행
FastAPI 백엔드를 기동하여 정적 파일과 API를 함께 서빙합니다.
1. 프로젝트 루트 폴더에서 uvicorn 백엔드를 가동합니다.
   ```bash
   python run_api.py
   ```
2. 이제 해당 PC에서는 `http://localhost:3700`으로 접속하여 모든 기능을 사용할 수 있습니다.

### Step 3. 다른 기기에서 접속하기
1. 서버를 실행한 PC의 사내 IP 주소를 확인합니다 (Windows 명령 프롬프트에서 `ipconfig` 입력 후 IPv4 Address 확인, 예: `192.168.1.150`).
2. 같은 Wi-Fi에 연결된 스마트폰, 태블릿, 혹은 동료 PC의 브라우저 주소창에 `http://192.168.1.150:3700`을 입력하여 접속합니다.
3. 실시간으로 현장의 작업자가 스마트폰으로 펀치 클락을 입력하면 사무실 모니터에 즉각 반영되는 모바일 환경을 테스트해 볼 수 있습니다.

## 2. 클라우드 배포 (대외 포트폴리오 데모용)

채용 담당자가 웹 링크를 클릭하여 어디서든 직접 접속해 볼 수 있게 클라우드 환경에 배포하는 방식입니다.

### 추천 플랫폼: Render.com (가장 간단한 무료/저비용 호스팅)

Render.com은 GitHub 리포지토리와 연동하여 자동으로 코드를 빌드하고 HTTPS가 적용된 고유 도메인을 발급해 주는 호스팅 서비스입니다.

#### 배포 절차:
1. **Render.com 가입**: 본인의 GitHub 계정으로 가입합니다.
2. **Web Service 생성**: Dashboard에서 **New +** -> **Web Service**를 클릭하고 연동된 GitHub 리포지토리를 선택합니다.
3. **환경 변수 및 빌드 명령어 설정**:
   - **Environment**: `Python` 선택
   - **Build Command**: 프론트엔드 빌드 및 백엔드 의존성 설치를 순차적으로 실행하는 쉘 스크립트를 빌드 명령어로 기재합니다.
     ```bash
     pip install -r requirements.txt && cd fe && npm install && npm run build
     ```
   - **Start Command**: uvicorn을 실행하여 서버를 가동합니다.
     ```bash
     python run_api.py
     ```
4. **SQLite 데이터 보존을 위한 디스크 볼륨 추가 (중요)**:
   - 클라우드 서버는 재배포 시 데이터가 초기화됩니다. 데이터 보존을 위해 Render의 **Advanced** 설정 내 **Disk Volume**을 생성하고 `data/` 폴더 경로를 마운트합니다 (예: Mount Path를 `/app/data`로 지정하여 SQLite db 파일을 보존).
5. **도메인 확인**: 배포가 성공하면 `https://steelworks-manager-react.onrender.com`과 같은 무료 고유 도메인이 생성되어 누구나 접속 가능한 데모 사이트가 완성됩니다.

이 클라우드 웹 링크를 이력서(CV) 및 포트폴리오 노션 페이지 상단에 추가해 두면, 채용 담당자가 언제든 브라우저를 켜고 직접 회원 가입, 작업 관리, 도면 업로드, QA Pass/Fail 처리를 누르며 당신의 IT 역량을 생생히 검증할 수 있게 됩니다.
