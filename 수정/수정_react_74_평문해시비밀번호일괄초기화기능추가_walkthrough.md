# 강제 12345678 평문 업데이트 및 테스트용 해시 일괄 업데이트 결과서

## 1. 조치 사항
- **백엔드 로직 분리 구현**:
  * `Reset to 12345678` (평문 초기화): 모든 임직원의 패스워드 값을 평문 텍스트 `'12345678'`로 저장합니다. 퀵필 노출 시 그대로 `'12345678'`로 노출되며, 로그인 요청 시 "비밀번호 변경 요청" 프로세스가 즉각 발동합니다.
  * `Reset to Hashed 12345678` (해시 초기화): 모든 임직원의 패스워드를 `bcrypt`로 암호화한 해시값으로 저장합니다. 퀵필에 해시 문자열이 그대로 노출되며, 해시 문자열 검증을 통해 로그인 요청 시 즉시 로그인 완료 처리(대시보드로 논스톱 진입)가 가능해집니다.
- **API 및 프론트엔드 연동 완성**:
  * 백엔드 및 훅 연동을 통해 `/api/admin/reset_passwords_hashed` 파이프라인 호출이 가능하도록 코드를 구성했습니다.
  * `Admin DB` 탭에 `Reset to 12345678` (주황색 아이콘) 및 `Reset to Hashed 12345678` (녹색 아이콘) 버튼 2개를 개별적으로 배치하여 선택 운용이 가능하도록 구성했습니다.
