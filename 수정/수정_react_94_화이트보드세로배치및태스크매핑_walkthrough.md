# 수정_react_94_화이트보드세로배치및태스크매핑_walkthrough

화이트보드(Whiteboard Task Board)를 직원 기준으로 수직 나열 및 스택 형태로 변경하고, 데이터 ID 바인딩 매핑 복구 및 퀵필터 확장 작업을 완료하였습니다.

## 구현 및 조치 상세 내용

### 1. 첫 컬럼에 직원을 두고 태스크를 세로로 스택 나열하는 세로형 레이아웃 도입
- [page.js (Whiteboard)](file:///f:/pe/public_html/steelworks-manager-react/fe/src/app/dashboard/whiteboard/page.js)
  - 기존의 가로형 나열 레이아웃을 첫 번째 고정 열(w-64)에 직원의 프로필, 직무, 작업 베이(Bay) 정보를 표기하고, 우측 열에 해당 직원의 태스크 카드들이 세로 방향(flex-col)으로 나열되는 **반응형 테이블 구조**로 수정했습니다.
  - 이를 통해 드래그 앤 드롭 재배치 및 가독성이 개선되었습니다.

### 2. 레거시 태스크 데이터 매핑 복구
- 분석 결과 마이그레이션된 `tb_tasks` 내의 `employee` 외래키 ID 값들이 과거 레거시 시스템 기준인 `1, 7, 11, 15, 18`번으로 지정되어 있었습니다.
- 현행 React 시스템의 실제 활성화된 직원 정보(`tb_login` 테이블 상의 `107 - Sam, 108 - Felipe, 111 - Teina, 115 - UserOffice, 117 - Usertruck`)와 매핑이 달라 데이터가 화면에 출력되지 않던 버그였습니다.
- DB 마이그레이션을 통해 해당 외래키 ID를 실제 임직원 고유 ID로 갱신 완료하였으며, 이에 따라 각 직원의 작업 행에 실제 데이터들이 정상 노출됩니다.

### 3. 'All Active Tasks (No Date)' 전체 검색 퀵 버튼 구현
- 날짜 필터링을 풀고 모든 만료일 이전의 활성 업무 카드를 일괄 브라우징할 수 있는 편리한 바로가기 링크 버튼을 `Quick Dev Filters (Data Available)` 패널의 최전방에 배치했습니다.

### 4. 컴파일 및 정적 빌드 이상 유무 검증
- `npm run build`를 통해 Turbopack 빌드 신택스 오류 없이 정상 배포 가능한 상태임을 최종 점검했습니다.
